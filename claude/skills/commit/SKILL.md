# Commit

## 手順

1. `git status` と `git diff` で変更の全体像を把握する
2. 変更を論理的なまとまりでコミット単位に分割し、各コミットのメッセージ案をユーザーに提示して確認を求める
3. ユーザーの確認後に `git add` + `git commit` を実行する

## コミットメッセージの書き方

大前提: **変更の重さに見合った簡潔さ**。短く済むものは短く書く。過剰な説明・場所の明示・不要な本文はユーザーに強く嫌われる。書く量は変更の粒度で決める。

### 1. 些末な変更 — 一語

書式整形・typo 修正・インデント調整・コメントの言い回し直しなど、diff を見れば意図も理由も自明なもの。変更の種類を表す一語だけにする。動機も場所も本文も書かない (`Co-Authored-By` は付ける)。

- 例: `書式整形`、`typo 修正`

判断を含む変更はここに入れない。例えば rename やファイル移動は「何を何に」は diff でわかっても「なぜその名前・場所か」が読めないことが多い。その場合は段階 2 で意図を 1 行書く (例: `リスト取得メソッドの命名を find_ 始まりに統一`)。純粋に機械的で理由も自明なときだけ一語でよい。

### 2. 通常の変更 — 1行で動機

「なぜ / 何のためにしたか」を1行で書く。**差分の列挙 (What) は書かない** — ファイル名・変更点の箇条書きは不可。差分から読み取れることは繰り返さない。

- 悪い: `○○メソッドにエラーハンドリングを追加`
- 良い: `○○処理が失敗したときにユーザーにエラーを伝えられなかった問題を修正`

### 3. 背景が要るとき — 本文を足す

**本文 (3行目以降) は原則不要**。1行目と diff から読み取れない非自明な背景・制約・「あえてこうしない理由」があるときだけ書く。その背景が README やコードに既にあるなら重複なので書かない。

### 禁止事項

- **差分の列挙 (What)**: 「何を変更したか」の羅列・箇条書き。上記 2 の通り。ただし段階 1 の一語 (`書式整形` 等、変更の種類名) はこれに当たらない
- **セッション・計画文脈に依存する語**: 「残りの」「先行変更に続き」「フェーズ2」など。前の PR や会話を知らないと通じない語は不可。各コミットは単体で意味が通ること (CLAUDE.md の付番禁止と同根)
- **ユーザーのプロンプト・個人ルールを指す語**: 「規約に合わせる」「ルール通りに修正」など。他人に通じない。何をしたかを具体語で書く
- **手動改行での折り返し**: 1文の途中で改行しない。72桁などで折り返さない (GitHub やエディタが表示幅に合わせて自動で折り返す)。1文が長すぎるとき (150文字を目安) は文を分割するか `- ` の箇条書きにする

Why の推論元として会話の文脈は使ってよいが、文脈依存の語をそのまま本文に残さないこと。文脈からも動機が不明なときはユーザーに確認する。

### 形式

```text
<1行目: 些末なら種類の一語、通常なら変更の動機・目的>

<3行目以降: コードから読み取れない背景・制約 (必要な場合のみ)>
```

- 1行目は「である調」 (レポート調) で書く
- 並列構造があるときは `- ` で始まるリスト形式にする

### AI エージェントとして実行している場合

粒度にかかわらず、コミットメッセージの末尾 (空行の後) に `Co-Authored-By` トレーラーを付与する。段階 1 の一語コミットでも同じ。モデル名はシステムコンテキストから読み取った実際のモデル名 (例: `Claude Opus 4.8`、`Claude Sonnet 4.6` など) を使うこと。

```text
Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
```

## 例

**例 1 — 些末な変更 (それぞれ別コミットのメッセージ全文。一語だけ):**

- `書式整形`
- `typo 修正`
- `コメント修正`

**例 2 — 通常の変更 (1行のみ。これが基本形):**

```text
○○処理が失敗したときにユーザーにエラーが伝わらなかった問題を修正
```

**例 3 — 背景説明が要るとき (例外的):**

```text
ユーザー増加による応答遅延を解消するためキャッシュを導入

ユーザー数増加により応答時間が3秒を超えるケースが発生。頻繁にアクセスされるデータを Redis でキャッシュすることで平均応答時間を 500ms 以下に短縮する。
```

**例 4 — AI エージェントが実行した場合 (末尾に `Co-Authored-By` を付与):**

```text
○○処理が失敗したときにユーザーにエラーが伝わらなかった問題を修正

Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
```

# Markdown 整形スキル

日本語混じりの Markdown 文書を、プロジェクト規約に従って整形する。

## ワークフロー

1. **入力を確認する** — ファイルパスが指定されていればそのファイルを Read で読み込む。インラインテキストが渡された場合はそのまま処理する
2. **整形ルールを順に適用する** — 下記の 3 つのルールを適用する
3. **変更結果を出力する**
   - ファイルの場合: Edit ツールで書き込み後、変更の要約を日本語で伝える
   - テキストの場合: 整形済みテキストをコードブロックで提示する

## 整形ルール

### 1. 日本語と英字の間のスペース

日本語文字 (ひらがな・カタカナ・漢字) と英字が隣接するとき、間に半角スペースを入れる。

**スペースを入れる対象**:

- 日本語 ↔ 英字のみ (`JavaScript`, `npm` など)
- 日本語 ↔ 英字+数字の混在 (`500ms`, `v1.2`, `macOS` など)
- 日本語 ↔ インラインコードのバッククォート

**スペースを入れない例外**:

- 日本語 ↔ 数字のみ (`10行`, `3つ`, `2倍` など)

**丸括弧のルール**:

- 必ず半角 `()` を使う
- `(` の前と `)` の後に半角スペースを入れる

例:

```text
× JavaScriptで書かれています
○ JavaScript で書かれています

× 応答時間は500msです
○ 応答時間は 500ms です

× 変数`hoge`を定義
○ 変数 `hoge` を定義

× 差分が10行程度    ← 数字のみなのでスペース不要
○ 差分が10行程度

× デフォルト値(100)を使う
○ デフォルト値 (100) を使う
```

### 2. 空行の挿入

構造要素の前後には空行を入れ、視覚的な区切りを明確にする。

空行を入れる位置:

- 見出し (`#`, `##` など) の**直後**
- リストの**前後**
- コードブロックの**前後**

例:

```text
× ## セクション
× 本文がすぐ続く行

○ ## セクション
○
○ 本文がすぐ続く行
```

### 3. コードブロックの言語指定

コードブロックには必ず言語識別子をつける。

- 言語が不明、または汎用的なテキストには `text` を使う
- コードブロックの中にコードブロックを含む場合は、外側を 4 つのバッククォートで囲む

例:

````text
× ```
× some code
× ```

○ ```python
○ some code
○ ```
````

## 注意事項

- 意味を変えるような編集はしない — スペース・改行の調整のみ行う
- 英語のみで書かれたテキストにはスペースルールを適用しない
- ユーザーが明示的に例外を指定した場合はそれに従う
